1. What is the difference between commit and push?

Commit saves changes to your local repository as a tracked snapshot with a description. Push sends those local commits to a remote repository such as Github to share or back up work.

3. What causes merge conflicts?

Merge conflicts occur when the same part of a file is modified differently in two branches being merged, and Git cannot automatically decide which change to keep. This includes conflicting edits on the same lines, mismatched file additions or deletions, or clashing formatting changes.
